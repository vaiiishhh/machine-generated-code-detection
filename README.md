# Revisiting Machine-Generated Code Detection and Authorship Attribution

> A comprehensive study of AI-generated code detection, LLM authorship attribution, and out-of-domain robustness across multiple programming languages, generators, and domains.

## Overview

Large Language Models (LLMs) such as GPT, CodeLlama, and Qwen have dramatically improved automatic code generation. However, this creates new challenges in identifying whether code is human-written or machine-generated and determining the source model behind generated code.

This project reproduces and extends the **CoDet-M4** framework for machine-generated code detection. We benchmark traditional machine learning methods and modern transformer-based models under multiple challenging settings, including unseen models, unseen domains, and unseen programming languages.

Our work also proposes a novel contrastive-learning-based approach for improving fine-grained authorship attribution.

---

## Key Contributions

- Implemented and evaluated **6 code detection models** across multiple experimental settings.
- Achieved **99.06% accuracy** on human vs AI-generated code detection using **UniXCoder**.
- Conducted extensive robustness analysis on:
  - Unseen code generators
  - Unseen software domains
  - Unseen programming languages
- Improved training stability of large transformer models through optimized fine-tuning strategies.
- Proposed a two-stage **contrastive representation learning pipeline** for LLM authorship attribution.

# Experimental Results

## 1. Binary AI Code Detection

**Task:** Classify whether a code snippet is written by a human or generated by an AI model.

| Model | Accuracy | F1 Score |
|---|---|---|
| UniXCoder | **99.06%** | **99.06%** |
| CodeT5 | 98.57% | 98.57% |
| CodeBERT | 98.26% | 98.26% |
| CatBoost | 92.79% | 92.78% |
| SVM | 87.22% | 87.22% |
| Fast-DetectGPT | 68.27% | 68.53% |

### Observation

Transformer-based models significantly outperform classical approaches because they learn deeper semantic and structural representations of code.

## Proposed Method: Contrastive Code Authorship Learning

Traditional classifiers struggle because different LLMs often generate code with highly similar styles.

To address this, we introduce a two-stage pipeline:

### Stage 1 — Contrastive Representation Learning

- Fine-tune a CodeT5 encoder using supervised contrastive loss.
- Learn embeddings that cluster code samples from the same author together while separating different generators.

### Stage 2 — Linear Classification

- Freeze the learned encoder.
- Train a lightweight linear classifier on top of the learned representation space.

### Performance

| Precision | Recall | Accuracy |
|---|---|---|
| 82.31% | 82.80% | 82.80% |

# Training Optimizations

During reproduction of the original CoDet-M4 experiments, the reported training configuration led to unstable transformer training.

We improved stability by:

- Reducing learning rate from `3e-4` to `2e-5`
- Reducing maximum sequence length from `512 → 256`
- Using larger effective batch sizes
- Applying gradient clipping
- Enabling mixed-precision training
- Selecting checkpoints based on validation F1 score

These modifications improved convergence and produced performance slightly higher than the original reported results.

# Key Findings

- Transformer-based models can detect AI-generated code with near-perfect accuracy under in-domain settings.
- Fine-grained authorship attribution remains difficult because modern LLMs share similar coding styles.
- Contrastive representation learning improves the separation of different code generators.
- Current detection systems are not robust to strong distribution shifts, especially unseen programming languages.
